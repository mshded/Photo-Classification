# Photo Classification

Проект для фильтрации изображений на веб-странице: пайплайн собирает все найденные картинки, отсекает технический мусор и оставляет только потенциально содержательные изображения.

## Что делает пайплайн

Для переданного URL пайплайн:

1. Парсит HTML и собирает кандидаты изображений (`img`, `srcset`, `picture > source`).
2. Скачивает кандидаты в локальную папку `data/raw/<page_id>/`.
3. Вычисляет метаданные (валидность, размеры, формат, размер файла и т. д.).
4. Применяет **hard prefilter** для явного мусора (tracking pixel, слишком маленькие изображения, служебные URL-сигналы и пр.).
5. Для оставшихся кандидатов считает ML-score и применяет порог из артефакта модели `models/best_model.pkl`.
6. Делает финальную дедупликацию и сохраняет только итоговые `final_keep` изображения.

Основной запуск выполняется через `run_demo.py`.

## Структура проекта

- `run_demo.py` — CLI-точка входа для запуска пайплайна на одном URL.
- `src/parser.py` — сбор кандидатов из HTML.
- `src/image_utils.py` — скачивание изображений и извлечение метаданных.
- `src/features.py` — генерация фичей и эвристики.
- `src/classifier.py` — обучение/загрузка модели и логика порога.
- `src/pipeline.py` — orchestration всего процесса и сохранение артефактов.
- `src/metrics.py` — расчет метрик.
- `data/labels.csv` — размеченный датасет.
- `notebooks/01_dataset_collection.ipynb` — сбор датасета.
- `notebooks/02_eda.ipynb` — EDA.
- `notebooks/03_training.ipynb` — обучение и подбор порога.

## Быстрый старт

### 1) Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2) (Опционально) Обучение модели

Если хотите переобучить модель, запустите ноутбук:

- `notebooks/03_training.ipynb`

После обучения должен появиться/обновиться файл:

- `models/best_model.pkl`

### 3) Запуск demo

```bash
python run_demo.py --url "https://example.com" --model_path models/best_model.pkl
```

Параметры CLI:

- `--url` — URL страницы (обязательный).
- `--output_dir` — папка артефактов demo (по умолчанию `results/examples`).
- `--raw_dir` — папка для сырых скачанных изображений (по умолчанию `data/raw`).
- `--model_path` — путь к артефакту модели (по умолчанию `models/best_model.pkl`).

## Артефакты запуска

Для каждого URL создается папка:

- `results/examples/<page_id>/`

Внутри:

- `page_info.json` — параметры запуска.
- `candidates.csv` — все кандидаты + признаки/решения фильтрации.
- `final_kept.csv` — только финально оставленные изображения.
- `run_log.json` — агрегированная статистика запуска.
- `final_keep/` — физические файлы итоговых изображений.

---

## Как устроена модель

- Модель: логистическая регрессия (scikit-learn pipeline).
- Есть этапы `train / val / test`.
- Порог вероятности подбирается на `val` с фокусом на precision.
- В inference используется сохраненный порог из `best_model.pkl`.

Также в проекте используется group-aware split, чтобы снизить утечки между train/val/test (через content hash / canonical URL и fallback-ключи).
